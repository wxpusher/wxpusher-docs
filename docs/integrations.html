如何选择推送模式

需求 建议
个人系统只通知自己 极简推送 SPT
多人接收、用户管理或 Topic 群发 标准推送
需要关注、付费状态或上行消息回调 标准推送

其他自动化系统

任何能发送 HTTP 请求的监控平台、CI/CD、NAS、Home Assistant、Shell、Python、Java 或 Go 程序，都可以使用 WxPusher。先用 SPT 验证通知链路，业务化或多人使用时再迁移到标准推送。
